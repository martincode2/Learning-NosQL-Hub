1.1 Identifying Database Requirements

Key Terms and Definitions

NoSQL (Not Only SQL)

A class of databases that go beyond the relational database model. Designed to handle large amounts of unstructured data and distributed systems. Uses non-relational data structures: documents, key-value stores, graph databases, wide-column stores.

MongoDB

Popular NoSQL database using document-oriented model. Stores data in flexible JSON-like documents. Features: automatic sharding, replication, and indexing.

Core MongoDB Concepts

Documents: Basic unit of data storage (JSON-like structures)

Collections: Groups of related documents (analogous to tables)

Indexing: Technique to improve query performance

Schema: Flexible structure that can evolve over time

Mongosh: Official MongoDB shell for command-line interaction

SQL vs NoSQL Comparison

Feature SQL NoSQL
Type Structured Query Language Not Only SQL
Data Model Relational database (RDBMS) Non-relational database
Schema Structured data with predefined schema Unstructured/semi-structured data
Data Storage Data stored in tables (rows/columns) Data stored in collections of documents
ACID Properties Follows ACID properties May not follow ACID properties
Query Support Supports joins and complex queries Uses denormalized data structures
Scaling Vertical scaling Horizontal scaling
Examples MySQL, PostgreSQL, Oracle MongoDB, Cassandra, Redis

Types of NoSQL Databases

Document-Oriented Databases

Store data as documents (JSON/BSON format) in collections.

Strengths: Flexible schema, supports nested structures, great for semi-structured data.

Examples: MongoDB, CouchDB, RavenDB.

Use case: E-commerce catalogs, content management, user profiles.

Key-Value Stores

Store data as key → value pairs.

Strengths: Very fast read/write, excellent for caching.

Examples: Redis, Amazon DynamoDB, Riak.

Use case: User sessions, shopping carts, leaderboards.

Column-Family Databases

Store data in rows and columns, grouped into column families.

Origin: Developed at Facebook and later open-sourced in 2008.

Purpose: Designed for massive, distributed datasets with high availability and fault tolerance.

Examples: Apache Cassandra, HBase.

Use case: IoT data, analytics, time-series data, real-time logging.

Graph Databases

Store data as nodes (entities) and edges (relationships).

Strengths: Best for relationship-heavy data.

Examples: Neo4j, Amazon Neptune, ArangoDB.

Use case: Social networks, recommendation systems, fraud detection.

Data Formats: JSON, BSON, and XML

JSON (JavaScript Object Notation)

Human-readable, text-based, lightweight. Commonly used in APIs and web applications.

{
  "name": "Alice",
  "age": 25,
  "skills": ["Python", "MongoDB"]
}

BSON (Binary JSON)

Binary-encoded form of JSON, used in MongoDB. Supports extra data types (e.g., Date, ObjectId). Faster and more efficient for storage/retrieval, but not human-readable.

XML (eXtensible Markup Language)

Text-based, uses tags to describe data. More verbose, widely used in legacy systems and configurations.

<person>
  <name>Alice</name>
  <age>25</age>
  <skills>
    <skill>Python</skill>
    <skill>MongoDB</skill>
  </skills>
</person>

1.3 Preparing Database Environment

MongoDB Scalability Factors

Horizontal Scalability:

  • Sharding: Distribute data across multiple servers
  • Automatic Sharding: Based on predefined sharding keys
  • Balanced Sharding: Even data distribution across shards
  • Indexing: Speed up query execution

Vertical Scalability:

  • Memory Optimization: Efficient memory usage
  • Query Optimization: Choose efficient execution plans

Additional Factors:

  • Replication: Data redundancy and fault tolerance
  • High Availability: Replica sets and sharded clusters
  • Cloud Integration: Available on various cloud platforms

Setting Up MongoDB Environment

Shell Environment Setup - Installation Steps:

  1. Visit official MongoDB website (https://www.mongodb.com/)
  2. Download appropriate version for your operating system
  3. Follow installation instructions
  4. Set environment variables (add MongoDB bin to PATH)
  5. Start MongoDB server: mongod
  6. Connect to MongoDB shell: mongo

Windows Installation Process:

  1. Download MongoDB installer
  2. Run setup file and click Next
  3. Choose Complete installation
  4. Select "Run services as Network service user"
  5. Optionally install MongoDB Compass (GUI)
  6. Complete installation
  7. Open Command Prompt
  8. Navigate to: C:\Program Files\MongoDB\Server\4.0\bin
  9. Run: mongod to start server
  10. Run: mongo.exe to start client

MongoDB Environments

Shell Environment (Mongosh)

Command-line interface for MongoDB. Direct database interaction. Suitable for developers and administrators.

Compass Environment

Official MongoDB GUI. Visual database management. User-friendly interface for non-technical users. Data visualization and query building tools.

Atlas Environment

Cloud-based MongoDB service. Automated deployment and scaling. Managed database solution. Global reach and high availability.
